用户配置

  • git config --global user.name 'sidewalk'
  • git config --global user.email '491187637@qq.com'
  • 在某个项目中配置不同用户信息则去掉global
  • 配置编辑器：git config core.editor vim

名词

  • HEAD：指向当前版本
  • reflog：命令历史
  • checkout：版本库替换工作区
  • reset：回退版本/把暂存区的修改回退到工作区
  • revert：反向提交

命令

  • 新建并切换分支：

    • git checkout -b feature/abc
    • git checkout -b bugfix/abc

  • 新建远程分支：

    • git branch feature/abc
    • git push -u origin feature/abc

  • 删除分支：

    • 本地：git branch -d feature/abc（-D为强制删除）
    • 远程：git push origin :feature/abc

  • 删除文件：

    • git rm a.txt
    • git commit

  • 与远程仓库建立关联：

    • git remote add origin git@github.com:michaelliao/learngit.git
    • git push -u origin master：推送至远程master，并把本地、远程master分支进行关联
    • git push origin master：之后推送

  • 合并：

    • git merge dev：把dev分支合并到当前分支
    • git merge --no-ff -m 'merge dev' dev：合并dev分支，不使用fast-forward模式

  • 暂存

    • git stash：暂存当前工作区
    • git stash list：查看暂存情况
    • git stash pop：恢复暂存

  • 远程库

    • git remote -v
    • git checkout -b dev origin/dev：创建远程分支到本地
    • git branch --set-upstream dev origin/dev：建立dev和origin/dev链接

  • 推送（分支）

    • git push origin master（指定本地分支）

  • 拉取

    • git pull --rebase：变基，不会产生一个merge commit

  • revert

    • git revert HEAD：撤销前一次commit，并产生一个新的commit
